# 🏨 리조트 예약 시스템 PRD

## 1. 개요
리조트 객실 예약을 위한 웹 기반 시스템을 개발한다.  
사용자는 회원 가입 후 로그인하여 객실을 조회하고 예약할 수 있으며, 예약 내역을 확인할 수 있다.

---

## 2. 주요 기능

- 회원 가입
- 로그인
- 객실 조회
- 객실 예약
- 예약 확인

---

## 3. 회원 (User)

### 3.1 회원 유형

| 유형 | 조건 |
|------|------|
| 분양 회원 | 분양회원번호 존재 |
| 일반 회원 | 분양회원번호 없음 |

### 3.2 회원 유형 결정 규칙

- 분양회원번호가 존재하면 **분양 회원**
- 분양회원번호가 없으면 **일반 회원**

---

## 4. 회원 가입

### 4.1 입력 항목

| 항목 | 필수 여부 |
|------|----------|
| 분양회원번호 | Optional |
| 이름 | Required |
| 연락처 | Required |
| 이메일 | Optional |
| 아이디 | Required |
| 비밀번호 | Required |

### 4.2 검증 조건

- 아이디 중복 체크
- 비밀번호:
  - 최소 8자리 이상
  - 영문 + 숫자 포함
- 필수 입력값 누락 검증
- 연락처 형식 검증
- 이메일 입력 시 형식 검증
- 분양회원번호:
  - 형식 및 유효성 검증 가능하도록 설계

---

## 5. 로그인

- 아이디 / 비밀번호 기반 로그인
- 로그인 실패 시 오류 메시지 제공

---

## 6. 객실 (Room)

### 6.1 데이터 제공 방식

- 초기 단계에서는 **Mock 데이터 기반**
- 관리자 등록 기능 없음 (조회만 제공)

### 6.2 객실 정보 항목

- 객실코드
- 객실명
- 재고
- 회원유형별 금액
- 예약 가능 회원유형

### 6.3 비즈니스 규칙

- 회원 유형에 따라 예약 가능 여부가 다름
- 회원 유형별 가격이 다름
- 객실 상세 페이지에서:
  - 예약 가능 여부 확인 가능
  - 회원 유형별 금액 확인 가능

---

## 7. 예약 (Reservation)

### 7.1 기본 조건

- 로그인한 사용자만 예약 가능
- 예약 단위:
  - **1박 1실만 가능**

### 7.2 입력 정보

- 투숙자 정보

### 7.3 검증 로직

예약 진행 시 반드시 다음 조건을 검증해야 한다:

1. 회원 유형이 해당 객실 예약 가능한지
2. 객실 재고가 존재하는지

### 7.4 재고 처리

- 예약 성공 시:
  - 해당 객실 재고 **1 감소**
- 재고가 0인 경우:
  - 예약 불가 처리

---

## 8. 예약 확인

### 8.1 접근 조건

- 로그인한 사용자만 접근 가능
- 본인 예약 내역만 조회 가능

### 8.2 제공 정보

- 예약번호
- 객실명
- 투숙자 정보
- 예약일자
- 회원유형
- 예약 상태

---

## 9. 페이지 구성

- 메인 페이지
- 회원 가입 페이지
- 로그인 페이지
- 객실 목록 페이지
- 객실 상세 페이지
- 예약 페이지
- 예약 확인 페이지

---

## 10. 권한 관리

| 페이지 | 접근 권한 |
|--------|----------|
| 예약 | 로그인 사용자만 |
| 예약 확인 | 로그인 사용자만 |
| 기타 일부 페이지 | 비회원 접근 제한 가능 |

---

## 11. 핵심 비즈니스 로직

- 분양회원번호 유무에 따라 회원 유형 결정
- 객실:
  - 회원 유형별 예약 가능 여부 존재
  - 회원 유형별 가격 차등 적용
- 예약:
  - 1박 1실만 가능
  - 회원 유형 및 재고 검증 필수
- 재고 처리:
  - 예약 성공 시 1 차감

---

- 취소/환불 정책
